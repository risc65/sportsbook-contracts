# Claude Context

This repository contains API contract definitions for a sportsbook system.

## Repository Purpose

- Store and version control gRPC protobuf definitions
- Store and version control OpenAPI/Swagger specifications
- Serve as the source of truth for API contracts across services

## Directory Structure

- `proto/` - gRPC .proto files
- `openapi/` - OpenAPI specification files (.yaml/.json)

## Guidelines

- Maintain backward compatibility when modifying existing APIs
- Use semantic versioning for breaking changes
- Follow protobuf style guide for .proto files
- Follow OpenAPI 3.x specification for REST APIs
